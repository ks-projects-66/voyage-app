# Source map

## Purpose

This document records how the standalone `voyage-app` repository maps back to the original proof-of-concept repository.

## Source repository

`ks-projects-66/le-grande-tour`

## Product source files

| Old path | New path | Status |
|---|---|---|
| `voyage/index.html` | `index.html` | Marketing page recreated in the new repo |
| `app/index.html` | `app/src/` | Ported to Vite+React compiled app |
| `supabase/functions/assistant/index.ts` | `supabase/functions/assistant/index.ts` | Present in repo (Gemini 2.5-flash, generalized for place/story/doc modes) |
| Supabase tables prefixed `wl_` | `supabase/migrations/` | To be codified as migrations |

## Current architecture

```text
voyage-app/
  index.html                    # Marketing page
  legal/                        # Privacy, terms, etc.
  docs/                         # Product specs and decision notes
  app/
    src/                        # React source (components/, lib/, styles.css, main.jsx)
    public/                     # Static assets and icons
    dist/                       # Built app (generated by npm run build)
    package.json
  supabase/
    migrations/                 # Database schema and RLS
    functions/
      assistant/                # Gemini Edge Function source
  ios/                          # Future Capacitor iOS project
```

## Migration principle

Do not keep Le Grand Tour and Voyage coupled. Le Grand Tour remains the personal trip proof-of-concept. Voyage becomes the production product.

## Ongoing work

Implement feature parity with Le Grand Tour (offline-first, live sync, AI capture) in the React app. See `docs/specs/2026-06-30-voyage-feature-parity.md`.
